Challenge

The Problem

Legacy rule-based systems failed to catch sophisticated fraud while generating 15% false positives, causing significant customer friction and operational overhead.

  • —15% false positive rate causing excessive manual reviews
  • —200ms+ latency delaying transaction processing
  • —Inability to adapt to new fraud patterns quickly

Solution

The Approach

Executed a multi-agent ensemble architecture combining XGBoost, LSTM, and rule-based models to balance accuracy, speed, and explainability.

01

Analyzed 6 months of transaction data for pattern discovery

02

Developed ensemble model architecture (XGBoost + LSTM)

03

Implemented real-time feature engineering with Kafka

04

Deployed on SageMaker with sub-50ms inference latency

Decisions

Why Ensemble?

Single models overfit. The ensemble captures rule-based patterns, feature interactions (Trees), and sequences (LSTM) simultaneously.

Why Kafka?

Required sub-100ms end-to-end latency. Kafka handles 50K+ TPS while maintaining strict ordering for session analysis.

Architecture

Kafka Ingestion → Real-time Feature Store (Redis) → Ensemble Inference (SageMaker) → Voting Logic → Decision API

Outcome

  • ✓Reduced false positives to 2.1%, significantly improving UX
  • ✓Achieved 45ms P99 latency for real-time blocking
  • ✓Saved $2.1M annually in direct fraud losses
  • ✓System handles 250K+ daily transactions with 99.9% uptime
